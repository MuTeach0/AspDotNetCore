# Assignment - M01 ModelAndInMemoryStoreSetup

الهدف: تحقق من فهمك للمشروع عبر الإجابة على الأسئلة التالية.

1) ما هي بنية `Product` (الخصائص الأساسية)؟

2) أين وكيف تُخزن البيانات عند التشغيل؟

3) اذكر اسم الملف الذي يحتوي على منطق CRUD.

4) كيف تضيف منتجًا جديدًا (اختصر الخطوات أو الطلب HTTP المطلوب)؟

5) ما الذي يجب تغييره لاستخدام قاعدة بيانات حقيقية بدلاً من التخزين في الذاكرة؟

---

اقتراح إجابات (راجعها وحرر إجاباتك):

1) `Product` عادةً تحتوي على: `Id` (int أو GUID), `Name` (string), `Price` (decimal), `Description` (string) (قد تختلف حسب التعريف الموجود).

2) تُخزن البيانات في `ProductStore` داخل بنية بيانات في الذاكرة (مثلاً `List<Product>`). البيانات مؤقتة وتُفقد عند إعادة التشغيل.

3) `Store/ProductStore.cs` يحتوي على منطق CRUD.

4) إرسال POST إلى endpoint المناسب مع JSON يمثل `Product`، أو تعديل الـ `request.http` وتشغيل الطلب.

5) استبدال `ProductStore` بخدمة تعتمد على EF Core أو أي مكتبة اتصال بقاعدة بيانات: إضافة `DbContext`, تكوين سلسلة الاتصال، واستبدال عمليات CRUD باستعلامات DB.
